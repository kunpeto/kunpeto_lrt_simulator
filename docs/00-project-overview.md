## 1. 專案目的

建立一個**網頁版淡海輕軌駕駛室操作模擬教學系統**，供司機員訓練使用。

本系統涵蓋兩大核心功能：

### 1.1 駕駛室設備操作模擬

以 3D 模型圖為背景，在圖片上疊加互動元素，模擬駕駛室內所有設備的操作：

- **設備互動**：43 項設備以圓點標示於精確位置，滑鼠懸停顯示名稱與說明
- **HMI 螢幕放大**：點擊 CMS / DCI / ATP / CCTV 螢幕時，放大進入全螢幕操作
- **啟動程序連鎖邏輯**：模擬 KEY ON → 自檢 → 行車模式 → 連鎖解除 → 安全煞車測試的完整流程
- **主控制器檔位模擬**：P → EB → B(0~100%) → N → D(0~100%) 連續可調
- **車速物理模擬**：不同檔位影響車速表指針動態變化
- **拖曳轉頭**：圖片寬度超過螢幕時，左右拖曳模擬駕駛室不同方向視角

### 1.2 故障排除模擬情境

提供各種故障情境的模擬訓練，讓司機員在安全環境中練習判斷與處置：

- **故障觸發**：模擬各類設備故障，CMS 顯示故障碼與警告
- **總故障燈判讀**：黃燈（警告）/ 紅燈（嚴重故障）的辨別與對應處置
- **故障排除流程**：依據故障碼查找原因，執行對應 SOP
- **降級模式操作**：ATP 故障、牽引故障等情境下的降級運轉操作
- **異常事件處理**：車門故障、集電弓異常、煞車系統異常等情境模擬
- **重開機程序**：WD0/WD1/WD2/WD3 各層級重開機流程

### 1.3 使用者回饋機制

提供內建的回饋功能，讓使用者（司機員、講師）可以直接在操作過程中回報問題或提出建議，開發端透過 Claude Code 讀取回饋進行改善：

- **回饋按鈕**：固定於畫面右下角，不遮擋操作
- **回饋類型**：`bug`（除錯）、`suggestion`（建議）
- **自動記錄上下文**：提交時自動附帶當前頁面/操作狀態
- **本機持久化**：回饋暫存於 localStorage
- **匯出供開發端讀取**：匯出為 `data/feedback.json`，Claude Code 可直接讀取

## 3. 技術架構

### 3.1 核心選型

| 項目 | 選擇 | 理由 |
|------|------|------|
| 部署方式 | **純靜態網頁** | 本機使用，不需伺服器，`file://` 可直接開啟 |
| 技術棧 | HTML + CSS + JavaScript | 零依賴，單機可運行 |
| 座標系統 | **SVG viewBox**（1411×613） | 對齊背景圖原始尺寸，座標不受縮放影響 |
| 圖層疊加 | SVG overlay | 透明 SVG 疊在背景圖上方，繪製所有互動元素 |
| 設備標記 | SVG circle + text | 圓點熱區 + 編號標籤 + Tooltip |
| HMI 螢幕 | 獨立面板彈出 | 點擊螢幕區域 → 彈出全螢幕面板 |

### 3.2 檔案結構

```
cab-simulator/               ← 模擬器本體（可獨立運行）
├── index.html               ← 主頁
├── css/style.css            ← 樣式
├── js/
│   ├── equipment-data.js    ← 設備清單與座標資料
│   └── cab-simulator.js     ← 主程式（渲染、互動、狀態機）
└── img/                     ← 圖片資源
    ├── cab-3d.png           ← 駕駛室 3D 模型圖（1411×613）
    ├── e-panel.svg          ← E 區應急面板向量圖
    └── cms-sprites/         ← CMS 螢幕切片圖
```

### 3.3 開發輔助工具

```
tools/                       ← 開發用標註工具（不隨模擬器發佈）
├── annotation-tool.html     ← 設備圓點座標標註
├── screen-annotation-tool.html ← 螢幕區域標註（四點/圓形）
└── cms-slice-tool.html      ← CMS 螢幕切片標註
```

### 3.4 資料檔案

```
data/                        ← 標註工具產出的座標資料
├── equipment-coordinates.json ← 43 項設備座標
├── screen-regions.json      ← 螢幕區域（四邊形/圓形）座標
└── cms-slices.json          ← CMS 元件切片座標
```
